# EquipmentAccountingApp
Этот проект моя учебная практика по РАЗРАБОТКА МОДУЛЕЙ ПРОГРАММНОГО ОБЕСПЕЧЕНИЯ ДЛЯ КОМПЬЮТЕРНЫХ СИСТЕМ в этом проекте я буду разрабатывать приложение согласно техническому заданию.
Проект разделен на этапы, каждый этап будет в отдельной папке.
ЗАДАНИЕ
на учебную практику

16.12	Создание БД и слоя доступа к данным (DAL).
1. Написать SQL-скрипты создания всех таблиц, первичных и внешних ключей, индексов (по инвентарному номеру).

17.12	Разработка и отладка бизнес-логики (BLL).
1. Обсуждение и утверждение архитектуры DAL, созданной асинхронно.
2. Добавить проект Class Library BLL.
3. Реализовать в BLL сервисные классы (например, DepartmentService, EquipmentService). Они инкапсулируют логику и используют Repository из DAL.
4. Фокус на отладке: Преподаватель дает задание с преднамеренной ошибкой в SQL-запросе или логике. Студент должен, используя инструменты отладки (брейкпоинты, Step Into/Over, Watch), найти и исправить ошибку.
5. Написать 2-3 простых юнит-теста (например, для проверки корректности добавления нового подразделения).	выполнено
18.12	Создание WinForms-интерфейса для модуля справочников.
1. Добавить проект Windows Forms App WinFormsUI.
2. Разработать главную форму (MainForm) с меню (MenuStrip) и панелью навигации (TreeView или ListBox).
3. Создать форму «Подразделения» (DepartmentForm):
- DataGridView для отображения списка.
- Кнопки "Добавить", "Изменить", "Удалить".

- Диалоговое окно редактирования (DepartmentEditForm).
4. Связать форму с DepartmentService из BLL. Реализовать загрузку данных, добавление, редактирование.
5. Валидация: Проверка на пустые поля в форме редактирования.
6. UI/UX: Сделать интерфейс аккуратным, использовать TableLayoutPanel для выравнивания, добавить иконки на кнопки.
7. Аналогично (по возможности) создать каркас формы "Сотрудники".	выполнено
19.12	Рефакторинг и работа над модулем «Оборудование». Групповой код-ревью.
1. Код-ревью (1 час): Преподаватель и студенты совместно смотрят код, написанный за асинхронный день. Обсуждаются лучшие практики, выявляются "запахи кода" (длинные методы, дублирование).
2. Рефакторинг (1 час): Студенты вносят правки в свой код на основе ревью: выделяют общие методы, улучшают именование.
3. Разработка сложной формы (2 часа): Коллективная работа над основной формой "Оборудование". Особое внимание:
- Использование ComboBox для выбора Типа, Сотрудника, Статуса (данные загружаются из БД).

- Реализация поиска/фильтрации по инвентарному номеру или сотруднику.
- Обсуждение и реализация паттерна "Наблюдатель" для обновления списка оборудования при изменении данных в справочниках.	выполнено
20.12	Завершение модулей и начало работы над отчетом.
1. Завершить форму "Оборудование" (CRUD, фильтрация).
2. Разработать форму "История перемещений". Реализовать логику: при смене ответственного в форме "Оборудование" автоматически создается запись в истории.
3. Реализовать модуль "Лицензии ПО" и "Установленное ПО" (связь многие-ко-многим). Это самая сложная логическая часть.
4. Инструментальная отладка: Задание – в коде формы "История" или "Лицензии" преподаватель оставляет скрытую ошибку (например, неверное условие в WHERE или утечка подключения). Студент должен:
- Используя профилирование SQL (SQL Server Profiler) или логирование запросов, найти неоптимальный запрос.
- С помощью трассировки или отладки по шагам найти логическую ошибку.

- Исправить и предоставить отчет об ошибке и ее решении.
5. Начать разработку простого отчета (форма "Отчеты" с кнопкой "Оборудование по подразделениям", вывод в DataGridView).	выполнено

критических методов сервисов (проверка корректности создания записи, проверка валидации).
4. Исправление найденных критических ошибок.	выполнено
